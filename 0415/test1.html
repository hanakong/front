웹을 통한 값의 전달방식

    웹에서의 서버와의 통신방식은 http를 사용한다.
    HTTP : Hyper Text  Transfer Protocol, 서버와 클라이언트 사이의 값들을 전송해주는 통신을 정의한 프로토콜(통신을 위한 규약)
    클라이언트가 서버에 요청(request)할 수 있고, 서버는 클라이언트에게 응답(response)해야한다.

    서버와 클라이언트의 자료 전달방식 : XML, JSON
    XML : 태그를 통한 전달방식
    
    AJAX : Asynchronous Javascript And XML - 비동기 통신
    XML : HTML과 같은 마크업 언어로, 초창기에 주로 사용했던 서버와 데이터를 주고받는 기술이다. // 동기식 통신이다.
          동기식 통신이며, 태그를 통해 전달하는데, 이때 불필요한 태그가 많이 전송되어 가독성이 떨어지는 단점이 있다.
          따라서, 요즘은 많이 사용되지 않는다.
    JSON : Javascript Object Notation으로 서버와의 필요한 데이터를 전송하는 포멧이다.
          초창기에는 모바일에서 주로 사용되었는데, 가볍고 빠른 장점 때문에 프로그램의 종류나 플랫폼에 상관없이 사용할 수 있게 되었다.
          자바스크립트에서는 객체와 같이 "키(key)":"값(value)" 으로 구성되어 있다.
          처리방식은 Object를 String으로 직렬화(serialize)처리해서 다시 Object로 변환하여 사용할 수 있다.
